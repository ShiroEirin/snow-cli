# Snow CLI VCPTUI 改造单

本文档用于固化本轮 `Snow CLI -> VCP TUI` 的最终改造依据，避免后续再次把 `Snow` 做成“前端解释后端 DSL”的错误方向。

## 一、定位

`Snow CLI` 的目标不是复刻 `VCPChat`，而是作为：

- `VCPToolBox` 的 `TUI / IDE agent front-end`
- 面向终端与开发工作流的轻量前端
- 对 `VCP` 协议做高保真透传、显示适配和最小必要 bridge

因此，Snow 的职责必须收敛为三类：

- `Outbound Request`：只补位 Snow 默认请求模型缺失的信息
- `Response Display`：只做协议显示兼容，不改写后端语义
- `Session Policy / Compression`：只做 VCP-aware 净化，避免把协议噪音重新喂回上下文

## 二、为什么要这样改

之前已经验证过两个问题：

1. 把 `::Time` 直接做成“注入历史消息”会污染请求体，偏离 `VCPToolBox` 的真实契约。
2. 把 `VCP` 特殊块直接挂到渲染热路径里做全文重写，会明显拖慢 `Snow` 输出。

对照 `VCPToolBox` 和 `VCPChat` 源码后的正确分层是：

- 协议真源看 `VCPToolBox`
- 显示/净化策略参考 `VCPChat`
- `Snow` 只迁移 `TUI` 真正需要的那一层

## 三、协议来源

### 1. 来自 VCPToolBox 后端

- `::Time` 由 `RAGDiaryPlugin` 从合并后的 `userContent + aiContent` 中解析
- `ROLE_DIVIDE_*` 由后端流式/非流式链路主动注入
- `TOOL_REQUEST` / `DailyNote` 是原生协议块
- `[[VCP调用结果信息汇总: ... VCP调用结果结束]]` 是 `vcpInfoHandler` 输出的工具结果块
- `[系统邀请指令:]`、`[系统提示:]无内容` 是后端已知的伪 user 前缀

### 2. 来自 VCPChat 前端

- 特殊协议块用独立 regex 解析，不和普通正文混着清洗
- 上下文净化时保留原始 VCP 协议意义，去掉无意义的包裹噪音
- `VCP元思考链` 作为特殊显示块处理，而不是普通正文

## 四、最终改造范围

### 1. 必做

- 保留现有 `::Time` bridge，继续只改写 outbound request
- 增加统一的 `VCP display parser`
- 在交互 UI 和无头模式里兼容：
  - `TOOL_REQUEST`
  - `VCP-ToolResult`
  - `DailyNote`
  - `ROLE_DIVIDE_*`
  - `VCP元思考链`
- 在 `contextCompressor` 中增加 `VCP-aware` 转录逻辑

### 2. 明确不做

- 不在 Snow 前端实现 `::Group / ::TagMemo / ::AIMemo / ::TimeDecay / ::RoleValve / ::Base64Memo`
- 不在 Snow 前端复刻 `vcpChat` 的富前端主动插件壳
- 不为了兼容去改写持久化 session
- 不把 VCP 兼容逻辑散落到 provider 层

## 五、四步实施

### Step 1. 文档固化

- 新增本文档
- 把“为什么这样改、哪些必须做、哪些不能做”写死

### Step 2. 协议显示层

新增统一 parser，负责解析：

- `<<<[TOOL_REQUEST]>>> ... <<<[END_TOOL_REQUEST]>>>`
- `[[VCP调用结果信息汇总: ... VCP调用结果结束]]`
- `<<<DailyNoteStart>>> ... <<<DailyNoteEnd>>>`
- `<<<[ROLE_DIVIDE_*]>>>`
- `[--- VCP元思考链 ---]`
- `<think>...</think>`

交互 UI 与 Headless 都走同一套解析结果。

### Step 3. 上下文压缩链

在 `contextCompressor` 中把 VCP 特殊块转成“对压缩模型有意义的简洁摘要”：

- 保留工具名、状态、核心内容
- 保留 `DailyNote` 的 `Maid / Date / Content`
- 折叠元思考链
- 去掉 `ROLE_DIVIDE_*` 这类纯协议噪音

### Step 4. 测试与回归

- 为 parser 和 transcript helper 增加单测
- 继续保留 `::Time` 与 `VCP元思考链` 的已有测试
- 至少跑一轮类型检查与差异校验

## 六、验收标准

满足以下条件才算本轮改造成立：

- `role.md` 中静态 VCP 语法继续原样透传
- `::Time` 续问 bridge 不回退
- `TOOL_REQUEST / TOOL_RESULT / DailyNote / 元思考链` 不再在 TUI 中裸显协议壳
- `ROLE_DIVIDE_*` 不再污染终端显示和压缩摘要
- `ghost anchor` 不被误删
- `contextCompressor` 不再把整段 VCP 协议噪音原样喂回下一轮

## 七、当前实现结论

本轮之后，Snow 对 VCP 的定位应当明确为：

- 一个保真透传 `VCPToolBox` 协议的 `TUI`
- 一个对 `VCP` 返回协议做结构化显示的终端前端
- 一个只在必要处补位、而不是篡改后端语义的 IDE agent

这才是 `Snow CLI` 作为 `VCP TUI` 的正确方向。
